# 音频生成策略文档

## 1. 目标

音频生成模块负责把章节文本稳定转换为可播放、可管理、可复听的音频文件。

目标：

- 生成稳定。
- 参数可控。
- 失败可重试。
- 输出结构清晰。
- 已支持 MP3 元数据写入和基础 M4B 打包，后续继续做播放器兼容性验收。

## 2. TTS 参数

Edge TTS 当前支持：

```text
voice
rate
volume
pitch
```

推荐暴露：

- 音色：下拉选择。
- 语速：慢速、正常、稍快、快速、自定义。
- 音量：较小、正常、较大、自定义。
- 音调：默认正常，高级设置。

推荐映射：

```text
慢速: -20%
正常: +0%
稍快: +15%
快速: +30%
```

## 3. 质量模式

### 快速模式

- 不做高质量转码。
- 不写歌词标签。
- 并发可稍高。
- 适合试听和快速生成。

### 标准模式

- 默认模式。
- 可做基础转码。
- 不强制写歌词标签。
- 适合大多数用户。

### 高质量模式

- 转码到更高码率和采样率。
- 可写入元数据。
- 速度较慢。
- 适合收藏。

## 4. 输出结构

默认输出：

```text
输出根目录/
  书名/
    001_章节名.mp3
    002_章节名.mp3
    转换日志.txt
```

成品化输出：

```text
输出根目录/
  书名/
    书名.m4b
    001_章节名.mp3
    002_章节名.mp3
```

## 5. 文件命名

默认模板：

```text
{index:03d}_{chapter_title}.mp3
```

规则：

- 清理非法字符。
- 限制长度。
- 重名时追加短编号。
- 保留序号，确保播放器排序正确。

## 6. 临时文件策略

所有临时音频文件必须创建在目标输出目录或目标文件同目录。

原因：

- Windows 不允许跨磁盘 `os.replace()`。
- 同目录替换更接近原子操作。
- 失败后便于清理。

清理规则：

- 成功替换后删除临时文件引用。
- 失败后尝试删除临时文件。
- 清理失败写日志，不中断整本书。

## 7. 重试策略

TTS 请求失败时按章节重试。

推荐：

- 默认重试 3 次。
- 使用指数退避。
- 单章失败后记录失败列表，继续后续章节。
- 转换结束后允许重试失败章节。

## 8. 长文本策略

长章节可能导致 TTS 超时或请求失败。

后续应支持：

- 按段落切分。
- 分段生成临时音频。
- 合并成单章 MP3。
- 分段失败可局部重试。

MVP 可以先记录长章节警告。

## 9. 背景音乐策略

背景音乐默认关闭。

启用时支持：

- 背景音乐目录。
- 背景音量。
- 是否循环。
- 每章随机或固定同一首。

产品建议：

- 背景音乐放高级设置。
- 默认音量不要超过 0.25。
- 转换前提示会增加处理时间。

## 10. 歌词标签策略

歌词标签是把章节文本写入 MP3 元数据，部分播放器可显示文本。

产品建议：

- 默认关闭。
- 放在高级设置。
- 文案说明“可能增加文件体积，部分播放器不支持”。

## 11. 元数据策略

短期：

- MP3 文件名体现章节。
- 日志记录书名作者。
- 写入 MP3 title、artist、album、track。
- 可选封面写入。

中期：

- 验收常见播放器的 MP3/M4B 元数据显示效果。
- 优化封面来源：支持自动提取 EPUB 内置封面。

长期：

- 支持更多 M4B 章节/封面兼容策略。

## 12. 验收标准

- 语速/音量/音调能传入 Edge TTS。
- 输出文件在书名目录内。
- 临时文件不跨盘替换。
- 单章失败不影响后续章节。
- 高质量转码失败会写日志。
- 背景音乐失败不会误删原音频。
- MP3 可写入章节标题、书名、作者、曲目号和封面。
- M4B 可按章节顺序生成，并写入章节标题元数据。
