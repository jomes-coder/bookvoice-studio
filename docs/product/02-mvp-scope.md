# MVP 范围文档

## 1. MVP 目标

MVP 的目标是把 EPUB 转 MP3 的主流程做成一个可靠的桌面产品闭环：

```text
选择 EPUB -> 自动识别书名和章节 -> 调整语音参数 -> 开始转换 -> 查看进度 -> 输出到书名目录
```

MVP 不追求支持最多格式，而是先把 EPUB 体验打磨到稳定、可理解、可恢复。

## 2. MVP 用户故事

### 用户故事 1：一键转换一本书

作为普通用户，我希望选择一本 EPUB 后点击开始，就能得到按章节拆分的 MP3 文件，并且输出在以书名命名的文件夹里。

验收标准：

- 用户选择输出根目录后，程序自动创建书名文件夹。
- 输出结构为 `输出根目录/书名/001_章节名.mp3`。
- 如果书名无法读取，则使用 EPUB 文件名。
- 文件夹名和文件名自动清理 Windows 非法字符。

### 用户故事 2：调整朗读效果

作为听书用户，我希望调整语速、音量、音调，让生成音频更适合自己的收听习惯。

验收标准：

- GUI 提供语速、音量、音调设置。
- CLI 提供 `--rate`、`--volume`、`--pitch`。
- 音色自检使用相同的语音参数。
- 日志记录本次语音参数。

### 用户故事 3：知道转换进度

作为用户，我希望知道当前转换到哪里、成功多少、失败多少、跳过多少。

验收标准：

- GUI 显示总章节、已完成、失败、跳过。
- 日志显示当前章节名和输出文件。
- 转换完成后显示结果摘要。

### 用户故事 4：失败后可恢复

作为用户，我希望某些章节失败后不用整本重跑。

验收标准：

- 已存在且非空的章节文件默认跳过；用户可开启覆盖重转。
- 失败章节在结果摘要中列出。
- GUI 提供“重试失败章节”的入口。

### 用户故事 5：章节不要明显混乱

作为用户，我希望转换出的章节顺序接近电子书目录顺序，不要把封面、目录、版权页读成正文。

验收标准：

- EPUB 优先按 spine 顺序解析正文。
- 可结合 toc 标题补全章节名。
- 默认过滤空章节、封面页、纯目录页。
- 章节预览中可以取消不需要的章节。

## 3. MVP 功能清单

### 必须包含

- EPUB 导入。
- 自动书名目录。
- EPUB 元数据识别。
- 章节顺序优化。
- 章节标题清理。
- 音色选择。
- 语速、音量、音调。
- 输出目录选择。
- 高质量/快速模式。
- 暂停/继续。
- 进度显示。
- 日志保存。
- 打开输出目录。

### 可延后

- PDF。
- 批量队列。
- M4B。
- 封面写入。
- 发音词典。
- 背景音乐高级混音。
- 配置云同步。

## 4. 推荐默认值

- 音色：`zh-CN-YunxiaNeural`。
- 语速：`+0%`。
- 音量：`+0%`。
- 音调：`+0Hz`。
- 并发章节数：`3`。
- 质量模式：标准。
- 歌词标签：默认关闭或放到高级设置。
- 高质量转码：标准模式开启，快速模式关闭。
- 背景音乐：默认关闭。

## 5. GUI MVP 页面结构

主界面分为四块：

- 导入信息：电子书路径、书名、作者、章节数。
- 转换设置：音色、语速、音量、音调、质量模式、输出目录。
- 章节列表：章节名、状态、是否转换。
- 任务状态：开始、暂停、继续、重试、进度、日志、打开输出目录。

## 6. 验收清单

- 选择一本正常 EPUB，可以生成书名目录。
- 书名目录下每章 MP3 文件序号连续。
- 文件名不会包含 Windows 非法字符。
- 语速设置会影响实际 Edge TTS 请求。
- 暂停后不再启动新章节，当前章节完成后等待。
- 输出目录在 GUI 中可直接打开。
- 转换结束有摘要。
- 日志文件记录输入文件、输出目录、音色、语速、质量模式、失败章节。

## 7. MVP 风险

- EPUB 元数据不规范，书名可能缺失。
- EPUB 章节结构差异大，toc 和 spine 可能不一致。
- Edge TTS 网络波动导致失败。
- 高并发可能触发服务限制。
- 长章节可能导致 TTS 超时。

## 8. MVP 成功标准

MVP 成功不是“支持很多格式”，而是用户能稳定完成一本 EPUB 的转换，并能清楚知道：

- 转了哪本书。
- 输出在哪里。
- 成功了哪些章节。
- 失败了哪些章节。
- 失败后怎么继续。
