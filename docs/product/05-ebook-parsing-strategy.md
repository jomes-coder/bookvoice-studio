# 电子书解析策略文档

## 1. 目标

电子书解析的目标不是简单抽取文本，而是产出稳定的“有声书章节模型”。

统一输出结构：

```text
Book
  title
  author
  language
  cover
  source_path
  chapters[]

Chapter
  index
  title
  text
  source_id
  word_count
  is_selected
  warnings[]
```

## 2. 格式支持优先级

### P0：EPUB

EPUB 是当前核心格式，结构化程度较高，适合作为第一阶段重点。

### P1：TXT

TXT 对小说用户价值高，切章规则可控，但编码和标题规则复杂。

### P1/P2：DOCX

DOCX 适合学习资料和文档，可通过标题样式识别章节。

### P2：PDF

PDF 是排版格式，不是天然文本结构。应作为实验支持，需要预览和手动修正。

### P2：MOBI/AZW3

实验支持。通过 Calibre `ebook-convert` 转换为临时 EPUB，再进入统一解析流程；DRM 加密文件不支持。

## 3. EPUB 解析策略

### 元数据读取

优先读取：

- title。
- creator/author。
- language。
- cover。

回退规则：

- title 缺失：使用文件名。
- author 缺失：显示“未知作者”。
- language 缺失：默认 `zh-CN` 或空。

### 章节顺序

优先级：

1. EPUB spine 阅读顺序。
2. toc 目录顺序辅助补标题。
3. 文件名自然排序作为最后回退。

spine 保证阅读顺序，toc 保证标题质量。两者不一致时，以 spine 为主，toc 只作为标题映射参考。

### 标题识别

标题优先级：

1. toc 中对应项标题。
2. HTML 中的 `h1/h2/h3`。
3. `title` 标签。
4. 文件名。
5. `第 N 章`。

标题清理：

- 移除换行和多余空格。
- 清理 Windows 文件名非法字符。
- 限制过长标题。
- 空标题使用 `第 N 章`。

### 正文抽取

策略：

- 使用 HTML parser 抽取正文。
- 优先读取 `body`。
- 移除 script/style/nav。
- 合并多余空白。
- 保留段落边界，用于未来控制停顿。

### 无效章节过滤

默认过滤：

- 空章节。
- 纯图片封面页。
- 目录页。
- 版权声明页。
- 字数过少且无有效标题的页。

谨慎过滤：

- 前言。
- 序章。
- 后记。
- 番外。

这些可能是正文，应默认保留但可在预览中取消。

## 4. TXT 解析策略

TXT 需要解决两个问题：编码识别和切章。

### 编码识别

优先尝试：

- UTF-8 with BOM。
- UTF-8。
- GBK。
- GB18030。

识别失败时提示用户选择编码。

### 切章规则

常见规则：

- `第1章 标题`
- `第一章 标题`
- `第 1 章 标题`
- `Chapter 1`
- `CHAPTER I`
- 独立标题行。

切章后需要预览，避免误切。

## 5. PDF 解析策略

PDF 不进入 MVP 核心质量承诺。

产品策略：

- 标注为“实验支持”。
- 必须先生成预览。
- 允许用户手动合并/拆分章节。
- 对扫描版 PDF 提示暂不支持 OCR，除非后续引入 OCR。

风险：

- 文本顺序错乱。
- 页眉页脚混入正文。
- 双栏排版错乱。
- 脚注干扰正文。

## 6. MOBI/AZW3 解析策略

MOBI/AZW3 不直接解析二进制结构，采用外部转换策略：

1. 定位 Calibre `ebook-convert`：用户配置路径、程序旁 `calibre/ebook-convert.exe`、系统 PATH、Windows 常见安装目录。
2. 将 MOBI/AZW3 转换为临时 EPUB。
3. 使用现有 EPUB 元数据、目录和章节解析链路。
4. 转换结束后删除临时 EPUB。

产品提示：

- 标注为“实验支持”。
- 环境检查显示 `ebook-convert` 是否可用；如果 GUI 填写了 Calibre 路径，优先检查该路径。
- 缺少 Calibre 时给安装提示。
- DRM 加密文件不支持，不做破解能力。

## 7. 章节预览

章节预览是提升识别精准度的关键功能。

每章显示：

- 序号。
- 标题。
- 字数。
- 预览前 200 字。
- 是否疑似无效章节。

用户动作：

- 勾选/取消。
- 修改标题。
- 合并相邻章节。
- 拆分章节。

MVP 可先支持勾选/取消，修改/合并/拆分后续再做。

## 8. 验收标准

- 常规 EPUB 可以按阅读顺序输出。
- 书名目录可从元数据或文件名生成。
- 空章节不会生成 MP3。
- 封面页和目录页默认不转换。
- 章节标题不为空。
- 文件名合法。
- 解析结果可用于 GUI 预览和转换。
- MOBI/AZW3 在安装 Calibre 且文件未加密时可进入同一预览和转换流程。
