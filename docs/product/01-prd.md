# PRD 产品需求文档

## 1. 产品概述

BookVoice Studio（书声工坊）是一款本地电子书转有声书工具。用户选择电子书文件后，产品自动识别书名、作者和章节结构，通过 Edge TTS 生成每章音频，并按书名整理输出结果。

产品当前已具备 CLI、桌面 GUI、音色选择、日志保存、暂停/继续、并发转换、高质量转码、歌词标签写入等基础能力。下一阶段的重点是把“能转换”升级为“稳定、清晰、可控地生产有声书”。

## 2. 产品定位

定位：本地有声书生产工具。

不是：

- 单纯的命令行转换脚本。
- 在线 TTS 平台。
- 电子书阅读器。
- 专业音频编辑软件。

是：

- 个人电子书转音频工具。
- 小说、学习资料、长文档的批量朗读生产工具。
- 可保留章节、书名、作者和输出结构的有声书整理工具。

## 3. 目标用户

### 普通听书用户

用户有 EPUB/TXT 小说，希望快速生成 MP3，在手机、车机或播放器里听。重点诉求是简单、稳定、输出位置清楚。

### 资深电子书用户

用户有大量电子书，关注章节准确、文件命名、音色、语速、批量转换、失败恢复和输出目录规范。

### 学习资料用户

用户希望把教材、讲义、长文章转换为音频，用于通勤、运动或睡前复习。重点诉求是可调语速、章节选择、内容过滤。

### 内容整理用户

用户关注书名、作者、封面、专辑、章节元数据，希望生成更接近正式有声书的音频资产。

## 4. 核心用户场景

### 场景 A：一本小说转 MP3

用户打开 GUI，选择一本 EPUB，保留默认音色和语速，点击开始。程序自动输出到 `output_audio/书名/001_章节名.mp3`，完成后用户点击“打开输出目录”。

### 场景 B：用户想加快朗读速度

用户选择音色后，把语速调整为“稍快”，试听一句话，确认后开始转换。日志记录本次使用的语速。

### 场景 C：转换中断后继续

用户转换到一半时关闭程序或网络失败。再次打开同一本书时，程序识别已存在章节，默认跳过已完成章节，并可重试失败章节；用户修改音色或语速后，也可以开启覆盖重转。

### 场景 D：章节识别不准确

程序解析出目录页、版权页和广告页。用户在章节预览中取消这些章节，再开始转换。

### 场景 E：批量处理多本书

用户把多本书加入任务队列，分别自动创建书名目录。任务队列按顺序转换，支持暂停、继续和失败重试。

## 5. 用户价值

- 节省时间：自动完成章节拆分和音频生成。
- 降低门槛：不用记命令，不用手动整理文件。
- 可控体验：音色、语速、音量、音调和质量模式可调。
- 稳定生产：失败可定位、可重试、可恢复。
- 资产管理：输出结构稳定，后续可导入播放器或同步到设备。

## 6. 功能范围

### 当前已有能力

- EPUB 转 MP3。
- 桌面 GUI。
- 音色下拉选择和刷新。
- 中文音色标签。
- 输出目录选择。
- 背景音乐目录选择。
- 并发章节转换。
- 高质量转码开关。
- 歌词标签写入开关。
- 暂停/继续。
- 每日日志文件。
- 语音自检。

### 下一阶段核心能力

- 自动按书名创建输出文件夹。
- 读取 EPUB 元数据：书名、作者、语言、封面。
- 更稳定的 EPUB 章节解析：spine/toc 顺序、无效页过滤。
- 语速、音量、音调设置。
- 转换进度：总数、完成、失败、跳过、当前章节。
- 打开输出目录。
- 失败章节一键重试。
- 章节预览与勾选。

### 中长期能力

- TXT/DOCX 支持，MOBI/AZW3 通过 Calibre 实验支持，PDF 后续实验支持。
- 批量转换队列。
- 配置预设。
- 文件命名模板。
- M4B 输出。
- 元数据和封面写入。
- 发音词典。
- 音量标准化。

## 7. 非功能需求

### 稳定性

- 单章失败不应导致整本书停止。
- 临时文件必须创建在目标目录同磁盘，避免 Windows 跨盘替换失败。
- 转换中断后可以通过跳过已存在文件恢复，必要时可覆盖重转。

### 性能

- 默认并发为 3，避免 Edge TTS 请求过多导致失败。
- 用户可调并发，但 GUI 应给出合理范围。
- 高质量转码和歌词标签应可关闭，以提升速度。

### 可理解性

- 普通用户看到中文摘要。
- 资深用户可以查看详细日志。
- 错误信息要说明原因和建议动作。

### 可维护性

- CLI 和 GUI 共享同一套转换核心。
- 电子书解析、音频生成、日志、GUI 状态应模块化。
- 新格式接入应通过统一的解析接口。

## 8. 关键指标

### 使用成功指标

- 用户首次使用能在 5 分钟内开始转换。
- 完成后能明确找到输出目录。
- 常见 EPUB 的章节顺序正确率达到可接受水平。
- 失败时用户能通过日志知道失败原因。

### 技术质量指标

- 单元测试覆盖核心解析、选项校验、输出路径、暂停控制和临时文件策略。
- 不依赖真实网络的测试可稳定运行。
- 大部分异常可在日志中定位到阶段：导入、解析、TTS、转码、标签、输出。

## 9. 替代方案和竞品视角

用户当前可能使用的替代方案：

- 在线 TTS 网站：上手快，但长文限制多，隐私和批量管理弱。
- Calibre + 插件或脚本：格式转换强，但 TTS 生产链路不够直接。
- 浏览器朗读/阅读器朗读：适合即时收听，不适合生成可管理音频文件。
- 专业音频软件：后期能力强，但文本到语音和章节管理门槛高。
- 自写脚本：灵活，但普通用户难以维护。

BookVoice Studio 的差异化应聚焦：

- 本地文件管理和可控输出。
- 章节级生产和失败恢复。
- 面向中文小说/资料的默认体验。
- GUI 和 CLI 同时服务普通用户与资深用户。
- 生成可收藏、可迁移、可播放的音频资产。

## 10. 分发策略

短期：

- 源码运行。
- README 和 GUI 启动脚本。
- 面向开发者和早期用户。

中期：

- Windows 免安装压缩包或安装包。
- 内置依赖检测。
- 首次启动引导。
- 示例 EPUB/TXT 测试文件。

长期：

- 版本发布说明。
- 自动更新或手动更新提示。
- 用户配置迁移。
- 插件化 TTS 服务扩展。

## 11. 商业化可能性

该项目短期更适合做个人效率工具和开源/半开源产品，不建议过早商业化。

潜在方向：

- 免费基础版：EPUB/TXT 转换、基础音色和语速。
- 高级版：批量队列、M4B、元数据、发音词典、多 TTS 服务。
- 定制版：企业文档转音频、内网部署、批量处理。

商业化前提：

- Windows 安装体验稳定。
- 常见电子书解析质量可靠。
- 失败恢复和日志诊断成熟。
- 明确 TTS 服务的使用边界和合规风险。

## 12. 边界和不做

短期不做：

- 商业在线平台。
- 账号体系。
- 云端同步。
- 专业音频剪辑。
- 全自动修复所有 PDF 版式问题。
- 复杂 DRM 解密。

PDF 不作为 MVP 核心卖点；MOBI/AZW3 标注为依赖 Calibre 的实验能力，不承诺 DRM 加密文件。
