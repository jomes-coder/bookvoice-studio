# 异常与日志策略文档

## 1. 目标

异常与日志策略的目标是让用户和开发者都能回答三个问题：

- 失败发生在哪个阶段。
- 为什么失败。
- 下一步应该怎么做。

## 2. 日志分层

### GUI 摘要日志

给普通用户看，强调动作和结果。

示例：

```text
正在解析电子书...
已识别《书名》，共 128 个章节。
正在转换 001_第一章...
转换完成：成功 126 章，跳过 2 章，失败 0 章。
```

### 详细日志文件

给排错使用，记录技术细节。

内容：

- 时间。
- 输入文件。
- 输出目录。
- 书名作者。
- 音色和语音参数。
- 并发、重试、质量模式。
- 每章状态。
- 异常信息。
- ffmpeg 失败原因。

## 3. 日志文件规范

当前已有每日日志：

```text
日志YYYYMMDD.txt
```

当前已增加随书转换清单：

```text
输出根目录/书名/conversion_summary.json
```

每日日志适合全局追踪，随书转换清单适合精确记录单本书的源文件、参数、统计结果和失败章节。

## 4. 阶段分类

所有错误应归类到阶段：

- 导入阶段：文件不存在、格式不支持、权限不足。
- 解析阶段：元数据缺失、章节为空、HTML 解析失败。
- TTS 阶段：网络失败、音色不可用、超时、服务限制。
- 后处理阶段：ffmpeg 失败、转码失败、背景音乐失败。
- 标签阶段：歌词/元数据写入失败。
- 输出阶段：目录权限、文件占用、跨盘替换。

## 5. 用户可理解错误

### 文件不存在

```text
错误：找不到电子书文件，请重新选择。
```

### 音色不可用

```text
音色自检失败：当前音色不可用，请刷新音色列表或更换音色。
```

### 网络失败

```text
语音生成失败：网络或 Edge TTS 服务暂时不可用，已重试 3 次。
```

### 输出目录不可写

```text
无法写入输出目录，请检查目录权限或更换输出目录。
```

### 高质量转码失败

```text
音频已生成，但高质量转码失败。可以关闭高质量模式后重试。
```

## 6. 失败处理原则

- 单章失败不终止整本书。
- 可恢复失败要提供重试入口。
- 不可恢复失败要阻止继续并说明原因。
- 后处理失败不应删除已生成的主音频。
- 清理临时文件失败只写日志。

## 7. 状态模型

章节状态：

- pending：待处理。
- running：处理中。
- skipped：已跳过。
- completed：已完成。
- failed：失败。

任务状态：

- idle：未开始。
- parsing：解析中。
- ready：可转换。
- running：转换中。
- paused：已暂停。
- completed：已完成。
- completed_with_errors：完成但有失败。
- failed：任务级失败。
- canceled：已取消。

## 8. 结果摘要

转换结束后必须输出摘要：

```text
书名：xxx
输出目录：xxx
总章节：128
成功：126
跳过：2
失败：0
耗时：01:23:45
```

如果有失败：

```text
失败章节：
- 023_第二十三章：TTS 超时
- 057_第五十七章：音频文件被占用
```

## 9. 诊断信息

详细日志应包含：

- Python 版本。
- edge_tts 版本。
- ffmpeg 路径。
- 输出磁盘。
- 任务参数。

这些信息有助于定位环境问题。

## 10. 验收标准

- 所有转换任务都有日志文件。
- GUI 中能看到转换摘要。
- 错误信息有中文说明。
- 失败章节可被定位。
- 跨盘临时文件问题不会再次出现。
- 日志不会只显示“未知错误”而没有上下文。
- 每本书输出目录都有 `conversion_summary.json`。
