# 版本路线图

## 总体路线

产品路线分为五个阶段：

1. EPUB 主流程产品化。
2. 语音参数和 GUI 体验升级。
3. 任务队列和失败恢复。
4. 主流格式扩展。
5. 成品有声书能力。

## V0.1 当前基础版

状态：已具备基础转换能力，且已继续补齐部分 V0.2-V1.0 底座能力。

已有能力：

- EPUB 转 MP3。
- CLI。
- 桌面 GUI。
- 音色选择。
- 中文音色标签。
- 日志保存。
- 暂停/继续。
- 并发控制。
- 高质量转码开关。
- 歌词标签开关。
- 跨盘临时文件问题修复。
- 自动书名输出目录。
- EPUB/TXT/DOCX 输入。
- 语速、音量、音调。
- 结构化转换结果摘要。
- GUI 失败重试入口。
- 环境检查命令。
- 取消任务。
- 批量任务队列。
- 首次启动引导。
- Windows 打包入口。
- MP3 章节标题/书名/作者/封面元数据。
- 整本 M4B 生成。
- M4B 章节标题元数据。
- MOBI/AZW3 实验支持，依赖 Calibre `ebook-convert`。
- 已存在章节默认跳过，并支持覆盖重转。
- 章节预览状态列，显示待转换、转换中、已完成、已跳过、失败、已取消。
- GUI 常用设置持久化，不保存当前电子书路径。
- 每本书输出 `conversion_summary.json` 转换清单。

主要不足：

- Windows 可执行包还需要在目标机器上实际执行 PyInstaller 并验收产物。
- M4B 的播放器兼容性还需要更多真实文件验收。
- MOBI/AZW3 需要用户安装 Calibre，且不支持 DRM 加密文件。

## V0.2 EPUB 产品化

目标：让一本 EPUB 的转换结果清晰、可控。

状态：核心能力已实现，章节预览和章节勾选已实现。

范围：

- 自动书名文件夹。
- EPUB 元数据读取。
- spine/toc 章节顺序优化。
- 无效章节过滤。
- 输出目录预览。
- 打开输出目录。
- 转换结果摘要。

验收：

- 常规 EPUB 输出到 `输出根目录/书名/`。
- 章节文件有稳定序号。
- 日志记录书名作者和实际输出目录。

## V0.3 语音参数和质量模式

目标：让用户控制朗读效果和速度。

状态：语速、音量、音调已实现；音色试听和背景音乐细调仍待做。

范围：

- 语速。
- 音量。
- 音调。
- 音色试听。
- 快速/标准/高质量模式。
- 背景音乐音量。
- GUI 高级设置折叠。

验收：

- CLI 和 GUI 参数一致。
- 自检与实际转换使用同一语音参数。
- 日志记录完整语音参数。

## V0.4 进度、失败重试和章节预览

目标：提升长任务可控性。

状态：进度摘要、失败重试入口、章节预览、章节勾选、章节状态列、取消任务和覆盖重转已实现。

范围：

- 进度条。
- 成功/失败/跳过统计。
- 章节列表。
- 章节勾选。
- 失败章节一键重试。
- 取消任务。
- 覆盖重转开关。

验收：

- 用户能看到每章状态。
- 失败后不用整本重跑。
- 章节预览可过滤明显无效页。

## V0.5 批量队列和 TXT 支持

目标：服务重度用户。

状态：TXT 支持和基础批量队列已实现；配置预设仍待做。

范围：

- 多本书任务队列。
- 队列暂停/继续。
- TXT 编码识别。
- TXT 自动切章。
- 配置预设。

验收：

- 多本书输出到各自书名目录。
- TXT 小说可按章节生成音频。
- 任务队列失败不影响其他任务。

## V0.6 DOCX/MOBI/AZW3/PDF 实验支持

目标：扩展学习资料和 Kindle 存量电子书场景。

状态：DOCX 基础解析已实现；MOBI/AZW3 通过 Calibre 转 EPUB 后复用现有流程；PDF 仍未实现，保持实验规划。

范围：

- DOCX 标题样式解析。
- MOBI/AZW3 通过 Calibre `ebook-convert` 转换为 EPUB。
- PDF 文本抽取实验支持。
- PDF 预览和手动修正提示。

验收：

- DOCX 常规标题可识别章节。
- MOBI/AZW3 缺少 Calibre 时有明确提示。
- PDF 明确标注实验能力。
- 用户可以在转换前预览 PDF 抽取结果。

## V0.7 有声书成品化

目标：让输出更接近正式有声书。

状态：第一版已实现 M4B 生成、封面写入和章节标题元数据；音量标准化、章节间停顿和播放器兼容性验收仍待继续。

范围：

- M4B 输出。
- 整本合并。
- 封面写入。
- MP3/M4B 元数据。
- 音量标准化。
- 章节间停顿。

验收：

- 生成可导入主流播放器的整本有声书。
- 章节信息可被支持的播放器识别。

## V1.0 稳定版

目标：面向普通用户稳定分发。

当前缺口：

- Windows 可执行包产物验收。
- M4B 真实播放器兼容性验收。
- 更完整的用户手册和示例文件。
- GUI 设置迁移和重置入口可后续补强。

范围：

- Windows 可执行包。
- 自动检查依赖。
- 稳定日志和诊断。
- 核心格式 EPUB/TXT/DOCX，实验格式 MOBI/AZW3。
- 完整 GUI 主流程。
- 基础用户文档。

验收：

- 非开发用户可安装运行。
- 无需命令行即可完成主流程。
- 常见失败有明确提示。

## 近期实施建议

已完成的近期顺序：

1. 自动书名文件夹。
2. EPUB 元数据和章节解析优化。
3. 语速/音量/音调。
4. 进度和打开输出目录。
5. 章节预览和失败重试。
6. 批量队列。

下一步建议：

1. 在目标 Windows 环境执行 `build_windows.bat` 并验收 `dist/BookVoiceStudio/`。
2. 用真实 EPUB/TXT/DOCX/MOBI/AZW3 样本验收 M4B、封面和章节元数据在常见播放器中的表现。

原因：

- 这些能力直接改善当前用户的实际问题。
- 不依赖大规模架构重写。
- 能形成清晰的产品体验提升。
