# GUI 交互设计说明

## 1. 设计目标

GUI 的目标是让用户不用理解 CLI 参数，也能稳定完成电子书到有声书的生产流程。

界面要避免把所有参数平铺给普通用户。常用设置直接展示，高级设置折叠。

## 2. 信息架构

主窗口建议分为四个区域：

```text
┌──────────────────────────────────────────────┐
│ 顶部：任务标题、当前书名、全局状态             │
├───────────────┬──────────────────────────────┤
│ 左侧设置区     │ 右侧工作区标签页               │
│               │                              │
│ 导入           │ 章节预览                     │
│ 语音           │ 批量队列                     │
│ 输出           │ 转换日志                     │
│ 高级           │                              │
├───────────────┴──────────────────────────────┤
│ 底部：开始/暂停/取消 + 重试/目录/环境/说明     │
└──────────────────────────────────────────────┘
```

当前实现中，右侧工作区使用“章节预览 / 批量队列 / 转换日志”三个标签页，避免长任务界面纵向堆叠。左侧任务按钮分为两行：第一行处理当前转换，第二行处理重试、输出目录、环境检查和使用说明。

## 3. 页面区域

### 导入区

字段：

- 电子书文件。
- 识别到的书名。
- 作者。
- 格式。
- 章节数。

交互：

- 点击“浏览”选择文件。
- 文件选择后自动解析元数据和章节。
- 解析失败时显示错误，不允许开始转换。

### 语音设置区

基础字段：

- 音色。
- 语速。
- 音量。
- 音调。
- 试听按钮。

推荐交互：

- 语速使用下拉或滑块：慢速、正常、稍快、快速、自定义。
- 音量使用下拉或滑块：较小、正常、较大、自定义。
- 音调默认折叠在高级设置。
- 试听使用固定文本或用户输入文本。

推荐默认值：

- 语速：正常 `+0%`。
- 音量：正常 `+0%`。
- 音调：正常 `+0Hz`。

### 输出设置区

字段：

- 输出根目录。
- 实际输出目录预览。
- 文件命名模板。

交互：

- 用户只选择输出根目录。
- 程序显示实际目录：`输出根目录/书名`。
- 如果目录已存在，提示将跳过已有章节或覆盖，取决于用户设置。

### 质量和高级设置区

字段：

- 质量模式：快速、标准、高质量。
- 并发章节数。
- 重试次数。
- 歌词标签。
- 背景音乐目录。
- 背景音乐音量。
- 覆盖重转。

默认：

- 质量模式：标准。
- 并发：3。
- 重试：3。
- 歌词标签：关闭或高级开关。
- 背景音乐：关闭。
- 覆盖重转：关闭。

### 章节列表区

字段：

- 是否转换。
- 序号。
- 标题。
- 字数。
- 状态。

状态：

- 待转换。
- 转换中。
- 已完成。
- 已跳过。
- 失败。

交互：

- 支持全选、取消全选。
- 支持过滤疑似无效章节。
- 支持失败后只看失败章节。

### 任务控制区

按钮：

- 开始转换。
- 暂停。
- 继续。
- 取消。
- 重试失败。
- 打开目录。
- 批量添加。
- 开始队列。
- 清空队列。

按钮状态：

- 未开始：开始可用，暂停/继续/重试不可用。
- 转换中：开始不可用，暂停可用，取消可用。
- 已暂停：继续可用，取消可用。
- 完成且有失败：重试失败可用。
- 完成且成功：打开目录可用。

### 批量队列区

用途：一次添加多本 EPUB/TXT/DOCX/MOBI/AZW3，按队列顺序转换。

展示字段：

- 序号。
- 状态。
- 文件名。
- 路径。

状态：

- 待转换。
- 转换中。
- 已完成。
- 失败。
- 已取消。

交互：

- 批量添加：选择多个电子书文件并去重加入队列。
- 开始队列：按顺序处理待转换任务，共用当前转换设置。
- 清空队列：未运行时清空所有队列项。
- 暂停/继续：作用于当前转换书籍，后续队列等待。
- 取消：当前章节收尾后停止当前书和后续队列任务。

## 4. 日志展示

界面日志不应直接堆所有技术细节。建议分两层：

摘要日志：

- 开始转换。
- 正在解析电子书。
- 共识别 N 个章节。
- 正在转换第 N 章。
- 完成/失败/跳过统计。

详细日志：

- Edge TTS 请求。
- ffmpeg 命令。
- 异常堆栈。
- 临时文件路径。

GUI 默认显示摘要，提供“查看详细日志”入口。

## 5. 关键文案

### 解析成功

```text
已识别《书名》，作者：作者名，共 128 个章节。
```

### 输出目录

```text
音频将保存到：D:\books_audio\书名
```

### 暂停

```text
已暂停：当前正在转换的章节会完成，后续章节等待。
```

### 完成

```text
转换完成：成功 126 章，跳过 2 章，失败 0 章。
```

### 失败

```text
有 3 个章节转换失败，可点击“重试失败章节”再次处理。
```

## 6. 可用性要求

- 所有路径输入都可通过浏览按钮选择。
- 所有错误都显示中文原因。
- 用户不用打开日志文件也能知道是否成功。
- 高级参数不影响默认一键转换。
- 转换中窗口不能卡死。

## 7. GUI 验收

- 选择文件后显示书名和章节数。
- 设置项不会重叠。
- 开始转换后按钮状态正确。
- 暂停后按钮变为继续。
- 完成后可以打开输出目录。
- 失败章节能在界面中看到。
