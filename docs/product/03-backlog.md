# 功能优先级 Backlog

## 优先级定义

- P0：下一阶段必须做，直接影响主流程可用性。
- P1：明显提升体验和效率，但不阻塞 MVP。
- P2：高级能力或格式扩展，适合在主流程稳定后做。
- P3：长期探索，不进入近期版本承诺。

## P0 Backlog

### 自动书名输出目录

需求：用户选择输出根目录，程序自动创建书名子目录。

价值：避免多本书音频混在一起。

验收：

- 输出为 `输出根目录/书名/001_章节名.mp3`。
- 书名来自 EPUB 元数据，缺失时使用文件名。
- 文件夹名清理非法字符。

### EPUB 元数据识别

需求：读取书名、作者、语言、封面。

价值：提升输出目录、日志、后续元数据写入能力。

验收：

- GUI 显示书名和作者。
- 日志记录书名和作者。
- 缺失元数据时有回退规则。

### EPUB 章节解析优化

需求：使用 spine/toc 优化章节顺序和标题。

价值：减少章节乱序、标题错误、无效页。

验收：

- 章节按阅读顺序输出。
- 过滤空章节、封面页、目录页。
- 标题优先来自 toc，其次来自正文标题。

### 语音参数

需求：支持语速、音量、音调。

价值：满足不同听书习惯。

验收：

- GUI 可设置。
- CLI 可设置。
- 自检和转换使用同一参数。
- 日志记录参数。

### 进度显示

需求：显示总章节、完成、失败、跳过、当前章节。

价值：减少用户等待焦虑。

验收：

- GUI 有进度条和数字统计。
- 转换结束显示摘要。

### 打开输出目录

需求：转换完成后可以一键打开输出目录。

价值：降低查找文件成本。

验收：

- Windows 下打开实际书名目录。
- 目录不存在时给出提示。

## P1 Backlog

### 章节预览与勾选

需求：转换前展示章节列表，用户可取消无效章节。

验收：

- 列表显示序号、标题、字数。
- 支持全选、反选、取消选中。
- 未选中章节不会转换。

### 失败章节重试

需求：失败后提供一键重试。

验收：

- 结果摘要列出失败章节。
- 重试只处理失败章节。
- 重试日志独立标记。

### 配置预设

需求：提供快速试听、小说朗读、高质量收藏等预设。

验收：

- 选择预设后自动填充参数。
- 用户仍可手动修改。

### 批量任务队列

状态：基础队列已实现。

需求：一次添加多本书，按队列转换。

验收：

- 每本书独立输出目录。
- 支持暂停队列。
- 每个任务有独立状态。
- 支持取消后停止后续队列任务。

### TXT 支持

需求：支持 TXT 小说导入和自动切章。

验收：

- 支持 UTF-8、GBK 常见编码识别。
- 支持“第 N 章”“Chapter N”等切章规则。
- 可预览和调整章节。

### 背景音乐参数

需求：支持背景音乐音量、是否循环、是否启用。

验收：

- 默认关闭。
- 开启时可调背景音量。
- 日志记录混音参数。

## P2 Backlog

### DOCX 支持

适合学习资料和办公文档，章节可通过标题样式识别。

### PDF 实验支持

PDF 应标注为实验能力，必须提供预览和手动修正，不承诺所有 PDF 精准。

### MOBI/AZW3 实验支持

状态：已通过 Calibre `ebook-convert` 接入。缺少 Calibre 时需要明确提示；DRM 加密文件不支持。

### M4B 输出

适合整本有声书，支持章节和元数据。

### 元数据写入

写入书名、作者、专辑、封面、章节信息。

### 发音词典

支持用户配置人名、术语、英文缩写的读法。

### 音量标准化

让不同章节响度更一致。

## P3 Backlog

- 云端任务同步。
- 多 TTS 服务供应商。
- 自动封面生成。
- 在线分享。
- 插件系统。
- 多语言 UI。

## 不做清单

近期明确不做：

- DRM 破解。
- 在线账号体系。
- 社区内容分发。
- 专业 DAW 级音频编辑。
- 自动保证所有 PDF 完美识别。
