# BookVoice Studio 使用说明书

本文档面向普通用户，说明如何用 BookVoice Studio（书声工坊）把 EPUB/TXT/DOCX 电子书转换成有声书音频，并说明 MOBI/AZW3 实验支持的使用条件。

## 1. 快速开始

Windows 下优先使用桌面界面：

```text
双击 start_gui.bat
```

如果使用命令行，也可以在项目根目录运行：

```shell
pdm gui
```

第一次打开后建议先点“环境检查”。如果 Python 依赖和 ffmpeg 都显示 OK，就可以开始转换。MOBI/AZW3 还需要 Calibre `ebook-convert` 可用；如果还没安装，点击“Calibre路径”旁边的“下载”；如果已安装但自动检测不到，可以在“Calibre路径”里选择 `ebook-convert.exe`。

桌面界面会自动记住上次使用的音色、输出目录、语速、音量、音调、质量选项、封面、背景音乐目录和 Calibre 路径；不会自动记住当前电子书路径。

## 2. 单本电子书转换

基本流程：

1. 点击“浏览”，选择 EPUB/TXT/DOCX/MOBI/AZW3 文件。
2. 在“章节预览”标签页确认章节识别结果和当前章节状态。
3. 双击不想转换的章节，把“转换”从“是”切换为“否”。
4. 选择音色、语速、音量、音调。
5. 选择输出目录。
6. 如果转换 MOBI/AZW3 且环境检查找不到 Calibre，在“Calibre路径”里选择 `ebook-convert.exe`。
7. 按需选择封面图片、背景音乐目录、M4B、元数据等选项。
8. 点击“开始转换”。

转换过程中可以使用：

- 暂停：当前章节会继续完成，后续章节等待。
- 继续：恢复后续章节转换。
- 取消：当前章节收尾后停止后续章节。
- 打开目录：转换后直接打开本书音频目录。
- 重试失败：只重试失败章节，已生成文件会自动跳过。

## 3. 批量队列

批量队列适合一次处理多本书。

操作方式：

1. 点击“批量添加”，选择多个 EPUB/TXT/DOCX/MOBI/AZW3 文件。
2. 在“批量队列”标签页查看队列状态。
3. 设置音色、输出目录、语速等通用参数。
4. 点击“开始队列”。

队列会按顺序一本一本转换。每本书都会自动输出到自己的书名文件夹。队列状态包括：

- 待转换。
- 转换中。
- 已完成。
- 失败。
- 已取消。

章节预览里的单章状态包括：

- 待转换。
- 转换中。
- 已完成。
- 已跳过。
- 失败。
- 已取消。

暂停和取消同样适用于队列任务。取消后，当前章节收尾，后续队列任务会停止。

## 4. 参数说明

常用参数：

- 音色：朗读声音。下拉框显示中文说明，括号内是真实 Edge TTS 音色 ID。
- 语速：控制朗读快慢，例如 `+0%`、`+15%`。
- 音量：控制朗读音量，例如 `+0%`、`+10%`。
- 音调：控制声音高低，例如 `+0Hz`、`+10Hz`。
- 重试次数：单章失败后的自动重试次数。
- 并发章节数：同时转换的章节数量。速度慢时可以提高；失败多时建议降低。
- Calibre路径：可选。用于指定 `ebook-convert.exe`，只影响 MOBI/AZW3。留空时会自动查找程序旁 `calibre/ebook-convert.exe`、系统 PATH 和常见安装目录。旁边的“下载”按钮会打开 Calibre 官方 Windows 64 位下载页。

成品选项：

- 高质量转码：转换到更高码率和采样率，速度会慢一些。
- 写入歌词标签：把章节文本写入 MP3 标签，部分播放器可显示文本。
- 写入 MP3 元数据：写入书名、作者、章节标题、曲目号和封面。
- 生成 M4B：额外生成整本有声书文件。
- 覆盖重转：重新生成已经存在的章节 MP3。默认关闭，关闭时已有章节会自动跳过。
- 封面图片：支持 JPG/JPEG/PNG，会写入 MP3 元数据和 M4B。
- 背景音乐目录：每章随机选择目录里的 MP3 作为背景音乐。

## 5. 输出文件说明

输出目录会自动按书名创建子文件夹：

```text
输出根目录/
  书名/
    001_章节名.mp3
    002_章节名.mp3
    conversion_summary.json
    书名.m4b
```

MP3 是每章一个文件，适合通用播放器。

M4B 是整本有声书文件，适合有声书播放器。支持的播放器通常可以记住播放位置，并显示章节。

元数据是写在音频文件里的书名、作者、章节标题、曲目号和封面。播放器能否显示这些信息，取决于播放器本身。

`conversion_summary.json` 是本次转换清单，记录源文件、书名作者、转换参数、成功/失败/跳过数量、失败章节和 M4B 路径。它用于排错，不影响音频播放。

## 6. 日志和排错

界面右侧“转换日志”标签页会显示当前运行日志。

日志也会写入项目根目录：

```text
日志YYYYMMDD.txt
```

常见问题：

- 没有音频文件：先看日志里是否有章节失败、音色自检失败或 ffmpeg 错误。
- 音色不可用：点击“刷新音色”，换一个中文音色后重试。
- 转换很慢：关闭高质量转码、歌词标签、M4B，或适当提高并发章节数。
- 失败很多：降低并发章节数，例如设置为 1 或 2。
- 修改音色或语速后没重新生成：勾选“覆盖重转”，否则已有章节默认跳过。
- 输出目录找不到：转换完成后点“打开目录”。
- 跨盘临时文件错误：当前版本会把临时文件放在输出文件同目录；如果仍出现类似错误，把输出目录换到项目同盘再试。
- MOBI/AZW3 失败：先点“环境检查”，确认 `ebook-convert` 可用；如果已安装 Calibre 但仍失败，手动选择 `C:\Program Files\Calibre2\ebook-convert.exe` 或把它放到程序旁的 `calibre` 目录；DRM 加密文件不支持。

## 7. Windows 打包

源码版可以直接双击 `start_gui.bat` 使用。

如果要生成可分发的 Windows 目录版程序，先安装 PyInstaller：

```shell
.venv\Scripts\python.exe -m pip install pyinstaller
```

然后运行：

```text
双击 build_windows.bat
```

生成结果位于：

```text
dist/BookVoiceStudio/
```

打包版默认不内置 Calibre。MOBI/AZW3 可以通过三种方式启用：用户本机安装 Calibre、在界面里选择 `ebook-convert.exe`、或把 `ebook-convert.exe` 放到程序旁的 `calibre` 目录。重新分发 Calibre 时需要单独遵守 Calibre 的 GPL 许可证要求。

## 8. 常见问题

### EPUB/TXT/DOCX 都支持吗？

支持。EPUB 支持目录和元数据解析；TXT 支持 UTF-8、GBK、GB18030 等常见编码；DOCX 支持基础标题样式切章。

### MOBI/AZW3 支持吗？

实验支持。程序会调用 Calibre 的 `ebook-convert` 先转成临时 EPUB，再进入现有章节解析流程。检测顺序是：界面填写的 Calibre 路径、程序旁 `calibre/ebook-convert.exe`、系统 PATH、Windows 常见安装目录。需要先安装 Calibre；DRM 加密文件不支持。

### PDF 支持吗？

当前不支持 PDF。PDF 文本抽取差异较大，后续会作为实验能力单独处理。

### 是否必须生成 M4B？

不必须。普通使用建议保留每章 MP3。需要整本有声书文件时再勾选 M4B。

### 背景音乐建议开启吗？

默认不建议开启。背景音乐会增加处理时间，也可能影响听书清晰度。

### 失败章节怎么处理？

转换结束后点击“重试失败”。程序会复用上次设置，只处理失败章节。

### 日志文件可以删除吗？

可以。日志只是排错记录，不影响已经生成的音频文件。

### 上次设置会保存在哪里？

GUI 设置会保存在当前用户的应用状态目录中。Windows 下一般是 `%APPDATA%\BookVoiceStudio\gui_settings.json`。删除这个文件后，界面会恢复默认设置。
